# MySQL Konfiguration
## Storages bei MySQL
Storage Engienes beschreiben wie die Daten gespeichert werden und werden pro Tabelle angewendet, in MySQL 8.x ist InnoDB als Standard definiert.
### Die 2 wichtigsten
* InnoDB
  * InnoDB ist die am häufigsten verwendete Speicher-Engine mit Transaktionsunterstützung. Es handelt sich um eine ACID-konforme Speichermaschine. Sie unterstützt die Sperrung auf Zeilenebene, die Wiederherstellung bei Abstürzen und die Steuerung der Gleichzeitigkeit mehrerer Versionen. Es ist die einzige Engine, die eine referenzielle Integritätsbeschränkung für Fremdschlüssel bietet.
* MyISAM
  * MyISAM ist die ursprüngliche Speicher-Engine. Es handelt sich um eine schnelle Speicher-Engine. Sie unterstützt keine Transaktionen. MyISAM bietet Sperren auf Tabellenebene. Es wird hauptsächlich im Web und im Data Warehousing eingesetzt.
### Andere Storage Engines
* ARCHIVE
  * Schnelle Inserts, hohe komprimierung
* BLACKHOLE
  * Speichert keine Daten alles geht in /dev/null
* MRG_MYISAM
  * Arbeitet mit MyISAM, fast mehrer MyISAM Tabellen logisch zusammen, gut für viele Daten
* FEDERATED
  * Erstellt eine logische Datenbank aus vielen Physischen Servern, abfrage immer auf dem Server wo die Tabelle ist, gut für Verteilte Server
* MEMORY
  * Speicher alles im RAM, ist sehr schnell, keine Datenspeicherung auf Festplatte
* CSV
  * Speichert alles in CSV hohe Felxibilität

### Überlegen Sie sich welche Daten mit welcher Storage Engine gespeichert werden könnten
* Blackhole
  * Testinserts
* CSV
  * In lokalen Client Anwendungen
* Memory
  * Temporäre Daten, evtl. essions
* Archive 
  * Archivedaten vom Bund, alte Daten über 10 Jahre
* MRG_MYISAM
  * Gut für Lagerhäuser oder Zalando
* Federated
  * Firma mit mehreren Standorten, jeder Standort hat eine Tabelle welche für Ihn ausgerichtet ist
* InnoDB
  * Banken (wird nur geschrieben wenn wirklick gemacht)
* MyISAM
  * Warenhäuser oder Webshops, Instagram

### Anwendung auf eien Tabelle
```sql
CREATE TABLE t1 (i INT) ENGINE = INNODB;
SET default_storage_engine=INNODB;
ALTER TABLE t ENGINE = InnoDB;
```

### Anwendungsbenutzer
```SQL
# Erstellt einen neuen User der von Überall zugreiffen kann
CREATE USER 'anwender'@'%' IDENTIFIED BY "Password123$";
# Ändert das Passwort
ALTER USER 'anwender'@'%' IDENTIFIED BY 'Hallo123$';
# Gitb die Rechte auf die Tabelle demo und alle Tabellen von überallher, besser wäre, wenn CREATE und DROP nur von localhost möglich wären
GRANT SELECT,INSERT,UPDATE,DELETE ON demo.* TO anwender@"%";
# Aktualisier die Rechte
FLUSH PRIVILEGES;
```

```sql
CREATE USER 'admin'@'localhost' IDENTIFIED BY 'ADmin123$';
# Gibt alle Rechte auf Demodatenbank

FLUSH PRIVILEGES;
SHOW GRANTS FOR 'admin'@'localhost';
```

## Seit MySQL 8.x
Das caching_sha2_password Plugin wird nun als Default für die Authentifizierung vernwendet, macht mit gewisser Software probleme
1. Im my.cnf ändern
```editorconfig
[mysqld]
default-authentication-plugin=mysql_native_password
```
2. Einstellung pro USer anpassen
```sql
CREATE USER 'demo'@'localhost'IDENTIFIED WITH mysql_native_password BY 'password';
```

#### Weiterführendes zu Mysql bei Ubuntu
Der Root User verwendet zum Authentifizieren das UNIX auth_socket Plugin. Nun gitb es 2 Optionen wie ich damit umgehen kann:
Option 1 root mit mysql_native_password authentifizieren:
```shell
$ sudo mysql -u root

USE mysql;
UPDATE user SET plugin='mysql_native_password' WHERE User='root';
FLUSH PRIVILEGES;
exit;

sudo service mysql restart
```
Option 2 neuen DB-User erstellen, welcher dem systemuser entspricht:
```shell
sudo mysql -u root # I had to use "sudo" since is new installation

USE mysql;
CREATE USER 'YOUR_SYSTEM_USER'@'localhost' IDENTIFIED BY 'YOUR_PASSWD';
GRANT ALL PRIVILEGES ON *.* TO 'YOUR_SYSTEM_USER'@'localhost';
UPDATE user SET plugin='auth_socket' WHERE User='YOUR_SYSTEM_USER';
FLUSH PRIVILEGES;
exit;

sudo service mysql restart
```
## Server-Konfiguration
### Optimierung
Die wichtigsten Parameter:
* key_buffer_size: Definiert die Anzahl Memory , welche für Tabellen-Indexe reserviert wird (Betrifft nur die MyISAM-Tabellen)
* innodb_buffer_pool_size Definiert die Anzahl Memory , welche für Tabellen-Indexe reserviert wird (Betrifft nur die InnoDB-Tabellen)
* table_open_cache Definiert die Anzahl gelichzeitg geöffneter Tabellen für alle Threads

Beispiele:
```shell
# 1-2GB Memory, viele Tabellen, wenige Clients
mysqld_safe --key_buffer_size=384M --table_open_cache=4000 \
           --sort_buffer_size=4M --read_buffer_size=1M &
           
# 256MB Memory, wenige Tabellen, viel Sortierungen
mysqld_safe --key_buffer_size=64M --sort_buffer_size=1M

# Sehr wenig Memory, viele Connections
mysqld_safe --key_buffer_size=512K --sort_buffer_size=100K \
           --read_buffer_size=100K &
```

### Netzwerk Config
Die IP Einstellungen und Hostname können ganz einfach über die GUI verwaltet werden.
Port öffnet man wie folgt:
```shell
ufw allow 53 # Für TCP und UDP
ufw allow 22/tcp # Nur TCP
ufw allow ssh # Öffnet den Port für den SSH Service
ufw allow from any to any proto tcp port 21,22 # Mehrer Ports
ufw allow from any to any proto tcp port 8080:8090 # Port range

# Spezifische Adressen
ufw allow from <Remote IP> to <Local IP> port <Port number> proto <Protocol>
ufw allow from 192.168.1.50 to any port 53
ufw allow from 192.168.1.50 to any port 22 proto tcp

# Für ein Netzwerk
ufw allow from 192.168.1.0/24 to any port 22 proto tcp

# Löschen
ufw delete allow 22/tcp
```
### Transaktions Isolation
```sql
SELECT @@GLOBAL.transaction_isolation, @@GLOBAL.transaction_read_only;
SELECT @@SESSION.transaction_isolation, @@SESSION.transaction_read_only;
```
![Hier sollte ein Bild über die Config erscheinen](../images/conf8.png)                              
Das bedeutet bei mir, dass Änderungen nicht zugelassen sind, solange eine andere Transaktion darauf zugreifft. Zudem wenn ein Index definiert ist wird nur die Spalte gesprerrt und ansonsten noch Lücken- und Nextkey sperren bei Update und Deletes, dies ist weil REPEATABLE.READ eingestellt ist. Asserdem ist eingestellt, dass bei nicht sperenden mehreren einfachen SELECTs die einen Bezug aufeinander haben auch konsisten sin.

## Config File Location
![Hier sollte ein Bild über die Configfile Location erscheinen](../images/conf5.png)             
Beispiel Config für /etc/mysql/mysql.conf.d/mysqld.cnf
```editorconfig
[mysqld]
# Required Settings
basedir                         = /usr
bind_address                    = 127.0.0.1 # Change to 0.0.0.0 to allow remote servers to connect to this server's
                                            # database instance
datadir                         = /var/lib/mysql
max_allowed_packet              = 256M
max_connect_errors              = 1000000
pid_file                        = /var/run/mysqld/mysqld.pid
port                            = 3306
skip_external_locking
skip_name_resolve
socket                          = /var/run/mysqld/mysqld.sock
#sql_mode                       = ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION,ONLY_FULL_GROUP_BY,STRICT_TRANS_TABLES # Enable for b/c with databases
                                                                                                                                            # using null dates
tmpdir                          = /tmp
user                            = mysql

default_storage_engine          = InnoDB
innodb_buffer_pool_instances    = 2     # Use 1 instance per 1GB of InnoDB pool size
innodb_buffer_pool_size         = 2G    # Use up to 70-80% of RAM & optionally check if /proc/sys/vm/swappiness is set to 0

key_buffer_size                 = 48M   # UPD
max_connections                 = 100   # UPD
table_open_cache                = 10000 # UPD


# Logging
log_error                       = /var/log/mysql/mysql_error.log
log_queries_not_using_indexes   = 1
long_query_time                 = 5
slow_query_log                  = 0     # Disabled for production
slow_query_log_file             = /var/log/mysql/mysql_slow.log
```

Das wichtige Darin

* log_error: Pfad der Log-Datei für Fehler
* datadir: Verzeichnis, in der die Daten gespeichert werden
* bind_address: Defaultmässig auf 127.0.0.1 - das bedeutet, dass man nicht aus dem Netzwerk auf den Server zugreifen kann. Um auf allen Interfaces zu hören, muss diese Einstellung auf 0.0.0.0 gesetzt werden.
* port: Der Default-Port von MySQL ist 3306.
* slow_query_log und slow_query_log_file: In Entwicklungsumgebung ist es interessant zu sehen, welche Datenbankqueries lange benötigen um erfolgreich beendet zu werden. Wenn Sie den Wert slow_query_log auf 1 setzen, können Sie über slow_query_log_file ein Log-File-Pfad eingeben
